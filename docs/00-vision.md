# 产品论点

## 为什么现有工具不够

现有周报/工作记录类工具在做**文档摘要**——把一堆文档压成几句话。但真正的痛点不是摘要，是**把散落的线索织成叙事**：这几周我到底推进了什么？卡在哪？接下来要做什么？摘要工具答不了这些问题。

## Trace 的底层模型

```
碎片捕获 → 证据（Evidence） → 工作线（Thread） → 成文汇报（Narrative）
```

核心原生概念是 **Thread（工作线）**。一条线跨多周、多素材、多决定、多人。它有状态（推进中 / 阻塞 / 完成）、有负责人、有开始时间、有相关证据。

这个抽象让两件事合二为一：
- 日常"随手记一笔"（Capture）
- 定期"写一份汇报"（Report）

它们本质是同一批线程的不同观察窗。

## 产品边界

- **不是**：项目管理工具（Jira/Linear 替代品）、知识库（Notion 替代品）、即时通讯
- **是**：个人工作上下文的本地归档 + AI 叙事生成器

## 灵魂设计

1. **证据必须可追溯**：每句话都能点回原始素材的某一段
2. **周期无关**：今天随手记，三个月后自动进季度复盘
3. **键盘优先**：⌘K / ⌘J / ⌘⇧N，鼠标党也能用
4. **AI 流式**：不转圈，字逐个冒出来
5. **本地优先**：数据全部 SQLite，用户自带 LLM Key
