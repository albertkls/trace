# PRD（Product Requirements Document）

## 一、信息架构

6 个一级模块 + 隐藏能力。

| 模块 | 做什么 |
|---|---|
| **Home / 今日** | 当日快记入口 + 热度线程 + 待关注 |
| **Inbox / 收件箱** | 原始素材待分拣（未归线程的证据） |
| **Threads / 工作线** | 看板 + 时间线 |
| **Notes / 记事** | Markdown 长文，按天归档 |
| **Todos / 待办** | 可勾选、可挂线程、可卷入周报 |
| **Reports / 汇报** | 任意周期报告库 + 编辑器 + AI 改写 |

隐藏能力：
- **⌘K** 全局命令面板
- **⌘⇧N** 系统级快记浮窗
- **⌘J** 选中文本呼出 AI 菜单
- **素材库 Sources**（审计回溯）
- **Settings**（LLM / 导出 / 集成）

## 二、核心流

### 流 1 · 5 秒快记
任何位置按 **⌘⇧N** → 浮窗打开 → 输入文字 → Enter 保存 → 自动挂当前活跃线程或落入 Inbox。

### 流 2 · 批量导入
拖一个文件夹到 Inbox → 逐一解析成证据 → AI 自动聚类到已有线程 or 建议新建线程。

### 流 3 · 写周报
打开 Reports → "新建 · W15 老板版" → AI 基于本周相关线程起草 → 用户在编辑器里改 → 导出。

### 流 4 · 线程维护
每周花 5 分钟扫一遍 Threads → 归档死线程 → 合并重复 → 新线程命名。

## 三、Report 成品形态

```markdown
# 2026-W15 周报（老板版）

## 本周综述
（2–3 句，高层视图）

## 本周重点推进

## 风险与待协调
**平台数据权限** 仍卡在合规审核…[4]

## 下周计划
…

---
## 证据附录
[1] 2026-03-03 项目A周例会 · 本周进展：完成接口联调…
[2] …
```

引用编号点击 → 跳回原始素材。

## 四、LLM 配置

- 走 OpenAI Chat Completions 兼容协议
- 字段：provider / base_url / api_key / model / temperature / max_tokens
- 预置 profile：OpenAI · DeepSeek · Kimi · 通义 DashScope · Claude（兼容网关）· Ollama
- 设置页有"测试连通"按钮

## 五、隐私边界（用户决策 2026-04-17）

- **不做本地脱敏**，全量发 LLM
- 用户自带 Key，风险自担
- 所有数据本地 SQLite，LLM 只是调用方

## 六、非功能要求

- **启动 < 1 秒**
- **AI 响应流式**（首 token < 2 秒）
- **离线可用**（不含 LLM 调用部分）
- **深色模式**
- **键盘可达率 100%**（每个功能都有快捷键）
- **可访问性**（WCAG AA）
