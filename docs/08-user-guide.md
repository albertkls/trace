# Trace 用户使用说明

本文档面向最终用户，介绍 Trace 是什么、适合谁、有哪些功能，以及每个功能怎么用。

---

## 1. Trace 是什么

Trace 是一个**本地优先的个人 AI 工作台**。  
它不是传统意义上的项目管理软件，也不是单纯的周报生成器。

Trace 更适合用来解决这类问题：

- 我这周到底推进了什么？
- 这些散落的记录、会议结论、风险和待办，怎么串成一条线？
- 我怎么快速写出一份能讲清楚来龙去脉的项目汇报？

Trace 的核心模型是：

```text
碎片捕获 → 证据（Evidence） → 工作线（Thread） → 成文汇报（Report）
```

在 1.1 版本里，Trace 还新增了：

```text
Project（项目） = 线程、记事、汇报的上层上下文
```

---

## 2. Trace 适合谁

Trace 适合：

- 同时推进多个主题的个人知识工作者
- 需要长期记录、阶段汇报、周期复盘的人
- 习惯边做边记，但又希望最后能沉淀成结构化成果的人

例如：

- 产品经理
- 研发工程师
- 设计师
- 独立开发者
- 创业者
- 需要写周报/月报/项目复盘的人

---

## 3. 核心概念先看懂

在开始使用之前，建议先理解 5 个核心概念。

### 3.1 Capture / 快记

你随手记下的一条内容，可能是一句话、一个风险、一个决定、一个待办线索。

例如：

- “权限联调已完成，等待验收”
- “老板想下周看一版项目综述”
- “埋点还缺导出字段”

### 3.2 Evidence / 证据

快记、导入文本或笔记中可被引用的工作事实。  
它是后续生成汇报的基础材料。

### 3.3 Thread / 工作线

一条持续推进的事情。  
比如：

- 权限系统重构
- 日本市场验证
- 用户增长实验

Thread 是 Trace 最核心的组织单位。

### 3.4 Project / 项目

Project 是 Thread 的上层上下文。  
一个项目下可以有多条线程、若干记事和多份汇报。

例如：

- Q3 增长实验
  - 注册流程优化
  - 新用户激活
  - 数据归因改造

### 3.5 Report / 汇报

在一个周期和受众视角下，把相关证据和线程整理成可读的文字输出。

例如：

- 本周项目报告
- 月报
- 复盘
- 1:1 对齐稿

---

## 4. 快速开始

### 第一步：配置 LLM

进入 **设置（Settings）**：

1. 新建一个模型配置
2. 选择 provider
3. 填入：
   - `base_url`
   - `api_key`
   - `model`
4. 点“测试”
5. 设置为默认

如果没有配置 LLM，你仍然可以记内容和整理线程，但无法使用 AI 生成摘要/报告。

## 5. 主界面功能总览

Trace 目前主要包含这些模块：

- 今日（Home）
- 收件箱（Inbox）
- 项目（Projects）
- 工作线（Threads）
- 时间线（Timeline）
- 记事（Notes）
- 待办（Todos）
- 汇报（Reports）
- 设置（Settings）

下面分别介绍每个模块。

---

## 6. 今日（Home）

### 它是干什么的

首页是你的轻量工作驾驶舱。

### 你可以在这里看到

- 当前需要关注的事项
- 草稿中的汇报
- 阻塞中的线程
- 活跃线程概览

### 你怎么用

适合每天打开一次，快速判断：

- 今天先处理什么
- 哪些线程在推进
- 哪些事情还没整理

---

## 7. 收件箱（Inbox）

### 它是干什么的

收件箱用来放暂时还没有归入线程的内容。

### 典型内容

- 一条闪记
- 一个风险提示
- 一个待整理的想法
- 某段临时记录

### 你可以做什么

- 修改分类（进展 / 决定 / 风险 / 计划 / 协同）
- 把它归入已有线程
- 新建线程并直接归入
- 转成待办
- 删除

### 你怎么用

推荐工作流：

1. 先随手记，不要犹豫放哪
2. 定期打开 Inbox 批量整理
3. 把真正重要的条目归入线程

### 推荐功能

Inbox 现在会根据内容**推荐项目**，帮助你更快判断它属于哪件事。

---

## 8. 项目（Projects）

### 它是干什么的

项目页是 1.1 版本新增的“上层上下文视角”。

它适合回答：

- 这个项目最近做了什么？
- 这个项目有哪些线程？
- 这个项目有没有待办、记事、报告？

### 项目列表页可以做什么

- 查看所有项目
- 按状态筛选
- 新建项目
- 编辑项目

### 项目详情页可以做什么

- 查看项目摘要
- 查看项目下的线程
- 查看项目下的记事
- 查看项目下的待办
- 查看项目下的报告
- 查看最近活动
- 在项目内搜索
- 新建线程 / 新建记事 / 新建项目报告
- 一键生成项目摘要
- 一键创建本周项目报告

### 项目时间线页

项目时间线会把以下内容放到同一条时间轴上：

- 证据
- 线程
- 待办
- 记事
- 汇报

适合用来回顾项目全过程。

### 推荐工作流

如果你在推进一件中长期的事，先建项目，再逐步往里面放：

- 线程
- 记事
- 汇报

---

## 9. 工作线（Threads）

### 它是干什么的

工作线是 Trace 的核心组织单位。

每一条 Thread 都代表：

- 一件正在推进的事
- 有明确主题
- 有自己的证据时间线
- 可挂待办
- 可生成报告片段

### 列表页可以做什么

- 查看所有线程
- 按项目过滤
- 查看状态
- 查看证据数量
- 新建线程

### 线程详情页可以做什么

- 查看线程证据时间线
- 编辑线程
- 合并线程
- 写成项目/汇报片段
- 查看关联待办
- 生成 AI 摘要
- 直接往该线程继续补记录

### 你怎么用

当你发现多条记录在描述同一件事时，就应该把它们组织到一个 Thread 中。

---

## 10. 记事（Notes）

### 它是干什么的

记事页用来写较长的 Markdown 文本。

适合记录：

- 项目思考
- 每日工作日志
- 会议后整理
- 复盘草稿

### 你可以做什么

- 新建笔记
- 自动保存
- 挂靠项目
- 挂靠多个线程
- 晋升为证据
- 按项目筛选

### 推荐功能

记事页现在会根据：

- 标题
- 正文
- 已挂靠线程

自动**推荐项目**。

如果挂靠线程和项目不一致，也会有轻提示。

### 你怎么用

如果你还不确定该建哪条线程，可以先写一条 Note，并先把它挂到某个 Project。

---

## 11. 待办（Todos）

### 它是干什么的

待办模块用来管理可以执行的事项。

### 你可以做什么

- 新建待办
- 勾选完成
- 设置截止时间
- 挂到线程
- 修改内容
- 删除

### 它和 Trace 其他模块的关系

- 待办可以从收件箱条目转化而来
- 挂在线程上的待办会自动成为项目的一部分（通过线程继承）

---

## 12. 汇报（Reports）

### 它是干什么的

汇报是 Trace 的最终输出模块。

### 你可以生成什么

- 周报
- 月报
- 项目报告
- 复盘
- 1:1 对齐稿

### 新建报告时你可以选择

- 周期范围
- 视角（老板 / 内部 / 1:1 / 复盘 / 自我）
- 项目
- 线程范围
- 模板

### 报告编辑器可以做什么

- AI 起草正文
- AI 改写
- 保存草稿
- 查看引用证据
- 最终定稿

### 项目报告能力

如果报告绑定了项目：

- 会自动使用项目上下文
- 模板会更偏项目叙事
- 可以直接从项目详情页一键创建本周报告

---

## 13. 时间线（Timeline）

### 它是干什么的

时间线把全局内容放回统一时间轴中查看。

### 会显示什么

- 工作线更新
- 收件箱条目
- 待办变化
- 笔记更新
- 报告更新

### 你怎么用

如果你想从“时间顺序”回顾而不是从“模块”回顾，就用 Timeline。

---

## 14. 设置（Settings）

### 它是干什么的

管理 LLM 配置。

### 你可以做什么

- 新建配置
- 编辑配置
- 删除配置
- 测试连接
- 切换默认模型

### 常见 provider

- OpenAI
- DeepSeek

- DashScope / 通义

## 15. 快捷键

### 已支持

- `⌘⇧N`：快速记录（Quick Capture）
- `⌘K`：全局搜索
- `Esc`：关闭当前弹层
- 某些表单支持 `⌘Enter` 快速提交

### 推荐使用方式

把 Trace 当成“随手记 + 结构化整理 + 最终输出”的闭环工具，快捷键会明显提升效率。

## 16. 推荐使用工作流

## 工作流 A：快速记录 → 整理 → 汇报

1. 用 `⌘⇧N` 记录想到的事
2. 内容先进入 Inbox
3. 定期整理到 Thread
4. 需要时生成 Report

## 工作流 B：项目推进

1. 先建一个 Project
2. 在项目下创建多个 Thread
3. 用 Note 记录项目思考
4. 在项目详情页看最近活动
5. 一键生成项目摘要和本周报告

## 工作流 C：复盘

1. 平时把证据和决策留在系统里
2. 到周末/月末时按项目或线程出报告
3. 在 AI 草稿基础上补充判断和总结

## 17. 常见问题

### Q1：没配 LLM 能用吗？

可以。

你仍然可以：

- 快记
- 整理线程
- 写记事
- 管理待办
- 手动写报告

但不能使用：

- AI 摘要
- AI 起草
- AI 改写

### Q2：项目和线程是什么关系？

- Project：一件事的整体上下文
- Thread：其中具体推进的一条线

一个项目下可以有多条线程。

### Q3：为什么有时候会提示项目和线程不一致？

因为：

- 你给 Note 选了一个项目
- 但又挂靠了属于其他项目的线程

这是允许的，但系统会提醒你检查归属是否合理。

### Q4：收件箱和记事有什么区别？

- Inbox：短内容、待整理、偏“临时信号”
- Notes：长文本、连续记录、偏“结构化思考”

---

## 18. 最后建议

如果你第一次使用 Trace，不必一开始就把所有内容都整理得很完美。

建议顺序是：

1. **先记录**
2. **再归类**
3. **最后输出**

Trace 的价值不在于“录入时就完全正确”，而在于它能帮助你把零散工作痕迹，慢慢整理成一条能讲清楚的叙事。
