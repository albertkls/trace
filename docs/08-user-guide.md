# Trace 操作手册

本文档面向 Trace 的实际使用者，介绍软件的用途、核心概念、主要功能和日常操作方法。当前内容按 Trace v1.9.0 整理。

---

## 1. Trace 是什么

Trace 是一个本地优先的个人工作记录与汇报工具。它的核心作用不是简单记笔记，也不是传统项目管理，而是把每天零散的工作痕迹整理成可追溯的工作线、项目上下文和最终报告。

Trace 的核心链路是：

```text
快记 / 文件 / 笔记 -> 证据 -> 工作线 -> 项目 -> 报告 / 导出
```

你可以把它理解成：

- 平时：快速记录进展、风险、决定、计划和待办。
- 整理时：把零散记录归入具体工作线和项目。
- 汇报时：基于真实证据生成周报、月报、项目报告或复盘。
- 归档时：把项目导出成 Markdown 包，离开软件也能阅读。

Trace 是本地优先应用，数据保存在本机 SQLite 数据库中。AI 相关能力只在你配置 LLM 后使用。

---

## 2. 适合什么场景

Trace 适合这些使用场景：

- 同时推进多个项目或主题。
- 工作过程里有很多碎片记录，需要事后整理。
- 需要定期写周报、月报、项目汇报、复盘。
- 需要保存“为什么这么做”的证据链。
- 想把本机文件、会议记录、Markdown 笔记和工作线索串起来。

典型用户包括：

- 产品经理
- 研发工程师
- 项目负责人
- 设计师
- 独立开发者
- 创业者
- 需要长期跟踪事务的人

## 3. 核心概念

### 3.1 工作区

工作区用于隔离不同领域的数据。每个工作区都有自己的项目、线程、证据、待办、记事、报告和设置。

常见用法：

- 一个工作区放正式工作。
- 一个工作区放个人项目。
- 一个工作区放测试数据。

在左上角工作区区域可以切换、新建或管理工作区。

### 3.2 快记

快记是最快的记录入口。你可以用它记录一句话、一条进展、一个风险、一个决定或一个计划。

快捷键：

```text
Command + Shift + N
```

快记可以直接进入收件箱，也可以直接挂到某条线程。

### 3.3 证据

证据是 Trace 中最重要的事实材料。它来自快记、Markdown 本地库、笔记晋升或导入内容。

证据可以有分类：

- 进展
- 决定
- 风险
- 计划
- 协同

报告生成、项目复盘、线程摘要都会依赖证据。

### 3.4 工作线

工作线是一件持续推进的具体事情。

例如：

- 二期调研：仓库和 WMS 收货差异
- 供应商初始化数据收集
- 登录流程改版
- 周报自动化

一条工作线下可以有证据、待办、关联文件和摘要。

### 3.5 项目

项目是工作线的上层上下文。一个项目可以包含多条工作线、记事、待办、报告和关联文件。

例如：

```text
SRM 二期
├── 仓库和 WMS 收货差异
├── 供应商初始化数据收集
├── 项目记事
└── 项目报告
```

项目页现在也是一个轻量驾驶舱，会显示健康度、周视图指标和下一步建议。

### 3.6 记事

记事适合写较长 Markdown 内容，例如会议纪要、复盘草稿、项目思考、每日记录。

记事可以挂靠项目，也可以挂靠多条工作线。重要段落可以晋升为证据。

### 3.7 待办

待办是可执行事项。它可以独立创建，也可以从收件箱证据转成待办。挂在线程上的待办会自然归入对应项目。

### 3.8 报告

报告是 Trace 的最终输出。它可以基于项目、线程、周期和受众生成。

报告支持：

- AI 起草
- AI 改写
- 引用证据
- 引用健康提示
- 导出 Markdown / 富文本 / `.md`
- 自动附引用清单

### 3.9 关联文件

关联文件用于把本机文件挂到项目、线程、证据、记事或报告上。

Trace 只保存文件路径和元信息，不移动、不复制、不修改原文件。关联后可以从 Trace 直接打开文件，或在 Finder 中显示。

## 4. 首次使用

### 4.1 打开应用

安装后打开 `Trace.app`。如果 macOS 提示未签名，首次可右键选择“打开”。

### 4.2 配置主题

进入设置页，可以选择：

- 跟随系统
- 浅色
- 深色

主题会保存到本机，下次打开 Trace 会自动沿用。

### 4.3 配置 LLM

进入设置页的 LLM 配置区域：

1. 新建配置。
2. 选择协议：OpenAI 兼容或 Anthropic。
3. 填写 `base_url`、`api_key`、`model`。
4. 设置温度和最大输出长度。
5. 点击测试连接。
6. 设置为默认配置。

没有配置 LLM 时，记录、整理、搜索、关联文件、备份、导出仍然可用，但 AI 摘要、AI 报告和 AI 改写不可用。

### 4.4 建议的第一次使用流程

1. 新建一个项目。
2. 新建一条工作线。
3. 用快记记录几条进展或风险。
4. 到收件箱整理未归档内容。
5. 进入项目详情页查看健康度和最近活动。
6. 需要汇报时新建报告并生成草稿。

## 5. 首页

首页是你的工作驾驶舱。

### 5.1 Focus Queue

Focus Queue 会提示当前最应该处理的事情，例如：

- 收件箱有多少条未整理记录。
- 哪些报告草稿需要定稿。
- 哪些工作线处于阻塞。
- 哪些项目沉默、阻塞或待汇报。

点击条目可以直接进入对应页面。

### 5.2 项目健康概览

首页会展示项目健康状态：

- 健康
- 活跃
- 阻塞
- 沉默
- 待汇报

每个项目卡片会显示：

- 下一步建议
- 本周新增证据
- 本周完成待办
- 本周活跃线程

### 5.3 昨日回顾

首页会显示昨天的活动，包括：

- 新增证据
- 完成待办
- 相关线程和项目

适合每天开始工作时快速回顾。

### 5.4 活跃工作线

首页会列出工作线活动情况，帮助你判断哪些事情近期推进最多、哪些事情卡住。

## 6. 快记

### 6.1 打开快记

你可以通过两种方式打开快记：

- 点击界面中的“写一笔”
- 使用快捷键 `Command + Shift + N`

### 6.2 写入内容

输入你要记录的内容。可以是：

- “数据收集完成，待整理后反馈”
- “客户要求下周前提供方案”
- “接口字段还缺 owner”
- “需要跟进验收材料”

### 6.3 选择分类

常用分类：

- 进展：已经发生的推进。
- 决定：已经明确的结论。
- 风险：可能影响结果的问题。
- 计划：下一步要做的事。
- 协同：需要他人配合的事项。

### 6.4 保存位置

快记可以：

- 保存到收件箱，稍后整理。
- 直接选择项目和线程，保存为该线程下的证据。
- 新建线程后直接归入。

如果选择了项目但仍保存到收件箱，这条记录暂时不会直接挂到项目；只有归入线程后，才会通过线程进入项目上下文。

---

## 7. 收件箱

收件箱用于集中处理未归档证据。

### 7.1 适合放什么

- 临时想法
- 快速记录
- 从 Markdown 本地库导入的材料
- 尚未决定归属的证据
- 需要批量整理的工作片段

### 7.2 单条整理

每条收件箱记录可以：

- 修改分类。
- 归入已有线程。
- 新建线程并归入。
- 转为待办。
- 关联文件。
- 打开或显示源文件。
- 删除。

### 7.3 批量整理

收件箱支持多选。选中多条记录后，可以批量执行：

- 归入同一线程
- 修改分类
- 转为待办
- 删除

### 7.4 按源文件分组

打开“按源文件”后，收件箱会按来源文件或来源类型分组，适合整理从本地 Markdown 库同步进来的大量记录。

### 7.5 推荐工作流

1. 日常先快速记录，不急着分类。
2. 每天或每周打开收件箱。
3. 按源文件分组查看。
4. 批量归入线程。
5. 把行动项转成待办。

## 8. 项目

项目是 Trace 的上层组织单位。

### 8.1 项目列表

项目列表会显示所有项目，包括：

- 项目名称
- 状态
- 线程数
- 记事数
- 报告数
- 健康度快照

### 8.2 新建项目

进入项目页，点击新建项目，填写：

- 名称
- 状态
- 负责人
- 摘要
- 颜色

### 8.3 项目详情

项目详情页包含：

- 项目摘要
- 基础统计
- 项目健康度仪表盘
- 工作线列表
- 报告列表
- 关联文件
- 记事
- 待办
- 最近活动

### 8.4 项目健康度

项目健康度根据项目下的数据自动计算。

指标包括：

- 阻塞线程
- 沉默线程
- 未完成待办
- 报告草稿
- 本周新增证据
- 本周完成待办
- 本周活跃线程

健康状态包括：

- 健康：暂无紧急动作。
- 活跃：有待办需要推进。
- 阻塞：存在阻塞线程。
- 沉默：线程长期没有活动。
- 待汇报：存在报告草稿。

### 8.5 项目内搜索

项目详情页有项目内搜索框，可以搜索：

- 工作线
- 记事
- 待办
- 报告
- 最近活动

### 8.6 项目常用操作

项目详情页可以直接：

- 编辑项目
- 生成项目摘要
- 新建记事
- 新建工作线
- 一键创建本周项目报告
- 新建项目报告
- 导出项目

### 8.7 导出项目

点击“导出项目”会下载一个 Markdown ZIP 包。

ZIP 包包含：

- `README.md`：项目概览、摘要、健康度、周指标。
- `threads/`：每条工作线的 Markdown 文件。
- `evidence.md`：项目全部证据。
- `todos.md`：项目全部待办。
- `notes/`：项目记事。
- `reports/`：项目报告。

导出只包含 Trace 中记录的文本和元信息，不会复制、移动或修改原始关联文件。

## 9. 工作线

工作线是一件具体推进中的事。

### 9.1 工作线列表

工作线列表可以查看：

- 标题
- 所属项目
- 状态
- 证据数量
- 是否置顶
- 最近活跃时间

### 9.2 新建工作线

可以从这些地方新建：

- 工作线列表
- 项目详情页
- 收件箱归档时
- 快记时

新建时可以选择项目，也可以从一条收件箱证据直接创建工作线。

### 9.3 工作线详情

详情页可以查看：

- 证据时间线
- 关联待办
- 关联文件
- 工作线摘要

可以执行：

- 编辑工作线
- 合并工作线
- 生成 AI 摘要
- 添加证据
- 添加关联文件
- 删除工作线

删除工作线时，会同步处理该工作线下的证据、待办和附件记录。

### 9.4 工作线状态

常见状态：

- active：进行中
- blocked：阻塞
- done：完成
- archived：归档

如果某条工作线卡住，建议改成 blocked，首页和项目健康度会自动提醒。

## 10. 记事

记事用于写较长的 Markdown 内容。

### 10.1 适合记录什么

- 会议纪要
- 项目思考
- 日志
- 复盘草稿
- 方案草稿
- 长段观察

### 10.2 新建记事

进入记事页，点击新建，填写标题和正文。正文支持 Markdown。

### 10.3 挂靠项目和工作线

记事可以：

- 挂靠一个项目
- 挂靠多条工作线

如果项目和工作线归属不一致，系统会提示你检查。

### 10.4 晋升为证据

当记事中的某段内容变成重要事实时，可以把它晋升为证据。晋升后它会进入收件箱或指定工作线。

### 10.5 关联文件

记事也可以关联本机文件，便于把会议录音、截图、方案文档和长文笔记放在同一上下文。

---

## 11. 待办

待办用于管理行动项。

### 11.1 新建待办

可以在待办页新建，也可以从收件箱证据转成待办。

字段包括：

- 内容
- 截止时间
- 所属工作线

### 11.2 完成待办

勾选待办后，会记录完成时间。本周完成待办会进入项目健康度和首页周视图指标。

### 11.3 修改待办

可以修改：

- 内容
- 截止时间
- 所属工作线
- 完成状态

### 11.4 待办和项目的关系

待办本身挂在线程上。线程属于项目时，待办自然出现在项目详情里。

---

## 12. 报告

报告是 Trace 的输出中心。

### 12.1 新建报告

可以从报告页新建，也可以从项目详情页新建项目报告。

新建时可以选择：

- 周期范围
- 受众视角
- 项目
- 工作线范围
- 标题

受众视角包括：

- 向上汇报
- 团队同步
- 1:1
- 复盘
- 自留

### 12.2 报告编辑器

报告编辑器包含：

- 左侧大纲
- 中间 Markdown 正文
- 右侧 AI 生成控制台
- 右侧证据引用面板
- AI 改写面板

### 12.3 AI 生成报告

配置好 LLM 后，可以点击“AI 生成”。

生成时可调整：

- 使用哪个模型配置
- 生成目标
- 篇幅
- 结构
- 重点
- 临时指令

生成完成后，Trace 会保存正文和引用证据列表。

### 12.4 引用证据

正文里用 `[n]` 表示引用第 n 条证据。

右侧证据面板会显示：

- 证据编号
- 分类
- 日期
- 所属线程
- 内容预览

你可以：

- 点击证据编号跳到正文引用位置。
- 点击“插入 [n]”把引用插到光标处。
- 拖拽证据卡片到正文插入引用。
- 打开所属线程或收件箱。

### 12.5 引用健康提示

报告编辑器会提示：

- 正文中出现无效引用。
- 某条引用证据已删除或不可访问。
- 有证据尚未在正文中引用。

这样可以避免报告里出现无法追溯的内容。

### 12.6 AI 改写

报告支持：

- 续写下周计划
- 压缩到精简版
- 换成其他口吻
- 自定义指令改写

改写结果会先以预览形式出现，确认后再应用到正文。

### 12.7 导出报告

报告支持：

- 复制 Markdown
- 复制为富文本
- 下载 `.md` 文件

导出时会自动附加“引用清单”，列出每个 `[n]` 对应的证据日期、项目、线程和原文。

## 14. 全局搜索

快捷键：

```text
Command + K
```

全局搜索可以搜索：

- 项目
- 工作线
- 证据
- 待办
- 记事

搜索结果遵守工作区隔离，不会跨工作区串数据。

搜索支持中文内容和前缀查询。点击结果可以跳转到对应页面。

## 15. 关联文件

关联文件用于让 Trace 记住本机文件的位置。

### 15.1 可以关联到哪里

目前可以关联到：

- 项目
- 工作线
- 证据
- 记事
- 报告

### 15.2 如何关联文件

在对应页面找到“关联文件”区域：

1. 点击“关联文件”。
2. 点击选择文件，打开系统文件浏览器。
3. 选择本机文件。
4. 保存关联。

### 15.3 关联后能做什么

关联后可以：

- 直接打开文件。
- 在 Finder 中显示文件。
- 移除关联。

移除关联只会删除 Trace 中的关联记录，不会删除原文件。

### 15.4 文件失效

如果原文件被移动、删除，Trace 会显示文件不存在或无法打开。原文件位置不会被 Trace 自动修改。

### 15.5 安全说明

Trace 对部分可执行或风险类型文件会限制直接打开，必要时只允许在 Finder 中定位，避免误执行。

## 16. 本地 Markdown 库

Trace 可以扫描本机 Markdown 文件，把内容同步到收件箱作为证据候选。

### 16.1 配置本地库

进入设置页：

1. 找到本地 Markdown 库。
2. 选择或填写目录路径。
3. 开启或关闭自动同步。
4. 点击扫描。

### 16.2 扫描结果

设置页会显示最近一次扫描结果：

- 扫描数量
- 新增数量
- 更新数量
- 未变化数量
- 移除数量
- 错误数量
- 错误明细

### 16.3 推荐用法

把已有 Markdown 工作记录目录接入 Trace，然后在收件箱按源文件分组，批量整理到工作线。

## 17. 设置

设置页包含多个区域。

### 17.1 界面主题

可以选择：

- 跟随系统
- 浅色
- 深色

主题设置会保存到本机。

### 17.2 软件更新

设置页可以检查更新。Trace 会通过 GitHub Release 获取最新版本。

当发现新版本时，可以下载并安装 DMG。更新前会自动创建数据库备份。

### 17.3 数据库备份

可以手动创建备份。备份保存在：

```text
~/Library/Application Support/Trace/backups
```

备份文件包含版本、时间和校验信息。

恢复备份时，Trace 会先创建当前数据库的安全快照。恢复失败不会覆盖当前数据库。

### 17.4 本地 Markdown 库

配置和扫描 Markdown 文件目录。

### 17.5 LLM 配置

管理模型配置：

- 新建
- 编辑
- 删除
- 测试连接
- 设置默认

支持 OpenAI 兼容协议和 Anthropic 协议，也可以接入自定义网关或本地模型服务。

## 18. 导出与归档

### 18.1 导出项目

项目详情页点击“导出项目”，会下载 Markdown ZIP 包。

适合：

- 项目归档
- 发送给他人
- 放入 Obsidian / Git / 网盘
- 做长期备份

### 18.2 导出报告

报告编辑器点击“导出”可以：

- 复制 Markdown
- 复制富文本
- 下载 `.md`

导出的报告会附引用清单。

### 18.3 数据备份和导出的区别

数据库备份：

- 用于恢复 Trace。
- 是 SQLite 数据库文件。
- 适合软件升级前和迁移前保存。

Markdown 导出：

- 用于阅读、归档、分享。
- 是普通 Markdown 文件。
- 不用于完整恢复 Trace 数据库。

## 19. 推荐工作流

### 19.1 日常记录

1. 用 `Command + Shift + N` 快记。
2. 能确定归属时直接选择项目和线程。
3. 不能确定时先放收件箱。
4. 每天结束前整理收件箱。

### 19.2 项目推进

1. 新建项目。
2. 按主题拆成多条工作线。
3. 每条进展、风险、决定都记录为证据。
4. 待办挂到对应工作线。
5. 首页看项目健康度。
6. 项目详情页看周视图和下一步建议。

### 19.3 周报生成

1. 平时持续留下证据。
2. 周末进入项目详情页。
3. 点击一键本周报告或新建项目报告。
4. 用 AI 生成草稿。
5. 检查引用证据。
6. 修改正文。
7. 导出 Markdown 或富文本。

### 19.4 会议纪要整理

1. 在记事页写会议纪要。
2. 挂靠项目和相关工作线。
3. 把关键结论晋升为证据。
4. 把行动项建成待办。
5. 把会议录音、截图、原始文档关联到记事或项目。

### 19.5 资料归档

1. 项目阶段结束后，进入项目详情页。
2. 点击导出项目。
3. 保存 ZIP 包到归档目录。
4. 如需可同时创建数据库备份。

## 21. 常见问题

### Q1：没有配置 LLM 可以用吗？

可以。记录、整理、搜索、关联文件、备份、导出都能用。不能使用 AI 摘要、AI 起草和 AI 改写。

### Q2：项目和工作线有什么区别？

项目是上层上下文，工作线是项目中的具体推进事项。一个项目可以有多条工作线。

### Q3：收件箱和记事有什么区别？

收件箱适合短内容和待整理信号。记事适合长文本和结构化思考。

### Q4：关联文件会复制原文件吗？

不会。Trace 只保存路径和元信息，不移动、不复制、不修改原文件。

### Q5：删除关联文件会删除原文件吗？

不会。只会删除 Trace 中的关联记录。

### Q6：导出项目会包含原始文件吗？

不会。项目导出只包含 Trace 已记录的文本和元信息。原始关联文件仍在原位置。

### Q7：备份和导出应该用哪个？

如果目的是恢复 Trace，用数据库备份。  
如果目的是阅读、分享、长期归档，用 Markdown 导出。

### Q8：搜索不到内容怎么办？

确认当前工作区是否正确。搜索只查当前工作区，不跨工作区。也可以尝试输入更短的关键词。

## 22. 使用建议

Trace 的最佳使用方式不是一开始就追求完美分类，而是先把工作痕迹留下。

推荐节奏：

1. 先记录。
2. 再整理。
3. 周期性生成报告。
4. 阶段性导出归档。

只要你持续留下证据，Trace 就能帮你把零散记录变成清晰的工作叙事。
