# Trace 优化路线图

本文档用于持续指导 Trace 的后续优化。排序原则是：先提升数据安全、核心闭环和大资料量下的可用性，再扩展自动化和智能化能力，最后做协作、插件化与更高级的体验增强。

## 优先级定义

- **P0 必做**：影响数据安全、发布更新、核心流程稳定性、资料增长后的性能。
- **P1 高价值**：明显提高日常效率，但不阻塞基础使用。
- **P2 增强**：扩大使用场景，或让产品更精致。
- **P3 探索**：方向有潜力，但需要更多验证。

## 当前基线

Trace 已具备这些核心能力：

- 本地优先桌面应用：FastAPI + React + pywebview + SQLite，macOS DMG 发布。
- 自动更新链路：GitHub Release 检查、DMG 下载、SHA256 校验、安装更新。
- 工作区隔离：项目、线程、证据、待办、记事、汇报按工作区过滤。
- 本地 Markdown 库挂载：可扫描 `.md` / `.markdown` 到收件箱，支持源文件定位、删除同步、启动自动同步。
- 项目与线程管理：项目、工作线、收件箱证据、待办、记事、汇报互相关联。
- LLM 配置：OpenAI 兼容协议、Anthropic 协议、模型连通性测试。
- 报告生成与改写：基于报告草稿和引用证据生成面向不同受众的 Markdown。

## P0 优化清单

### 1. 检索性能与资料可发现性

目标：当本地库、收件箱、笔记和线程持续增长后，`⌘K` 搜索仍能快速、稳定、跨模块找到内容。

实施方向：

- 使用 SQLite FTS5 建立统一全文索引。
- 覆盖项目、线程、证据、待办、记事。
- 搜索结果按相关性优先，再按时间兜底。
- 继续保留工作区隔离。
- 后续可增加命中高亮与搜索历史。

验收标准：

- 搜索项目、线程、证据、待办、记事均可命中。
- 同一关键词在不同工作区不会串数据。
- 后端测试覆盖 FTS 与中文前缀查询。

### 2. 数据备份与恢复

目标：任何版本升级或本地库同步前，用户都能安全回滚数据。

实施方向：

- 设置页增加“立即备份数据库”。
- 自动更新前自动创建备份。
- 支持从备份恢复，需要二次确认。
- 备份文件命名包含版本、时间和数据库校验信息。

验收标准：

- 备份文件可在 `~/Library/Application Support/Trace/backups` 找到。
- 恢复后工作区、项目、线程、证据完整。
- 恢复失败不会覆盖当前数据库。

### 3. 本地库同步队列与状态可见性

目标：本地库自动同步更可控，用户能看到同步是否成功、失败在哪里。

实施方向：

- 记录每次同步批次、耗时、错误数量。
- 设置页展示最近一次同步结果。
- 对大文件、不可读文件、编码问题给出可读错误。
- 后续可升级为后台周期同步或文件系统监听。

验收标准：

- 最近一次同步结果可持久化显示。
- 同步错误不会中断整个批次。
- 自动同步可按工作区开启或关闭。

## P1 优化清单

### 4. 收件箱批量整理

目标：降低本地库导入后大量证据的整理成本。

实施方向：

- 收件箱支持多选。
- 批量归入同一线程。
- 批量改分类、删除、转待办。
- 增加“按源文件折叠”视图。

验收标准：

- 可一次处理多条证据。
- 批量操作后项目、线程、收件箱计数刷新正确。

### 5. 报告引用质量提升

目标：让生成报告更可信、更可追溯。

实施方向：

- 报告编辑器展示引用证据详情。
- AI 生成时要求每段输出绑定证据 ID。
- 报告导出时附“引用清单”。
- 引用缺失或已删除时给出警示。

验收标准：

- 报告中引用的证据可点击回到来源。
- 删除证据后报告能显示缺失引用。

### 6. 项目健康度与周视图

目标：让首页和项目页更像“工作驾驶舱”。

实施方向：

- 项目健康度：活跃、阻塞、沉默、待汇报。
- 周视图：本周新增证据、完成待办、活跃线程。
- 首页按优先级推荐下一步整理动作。

验收标准：

- 用户打开首页可以快速知道该处理什么。
- 项目页能看到趋势和阻塞风险。

## P2 优化清单

### 7. 导出与迁移

实施方向：

- 导出项目为 Markdown 包。
- 导出报告为 Markdown / PDF。
- 导入 JSON 备份。
- 为未来移动端或 Web 端保留数据迁移格式。

### 8. 模板系统

实施方向：

- 报告模板：周报、月报、复盘、1:1。
- 线程模板：项目启动、实验跟踪、事故复盘。
- 快记模板：决策、风险、进展、计划。

### 9. 本地模型与隐私增强

实施方向：

- Ollama 体验优化。
- 每个工作区绑定默认模型。
- 敏感工作区禁止调用云端 LLM。
- AI 调用前显示将发送的上下文摘要。

## P3 探索清单

### 10. 文件系统监听

使用 watchdog 或 macOS FSEvents 监听本地库变化，替代启动扫描和手动扫描。

### 11. 插件化数据源

接入日历、邮件、GitHub、Linear 等数据源，把外部事件转成证据。

### 12. 多设备同步

探索 iCloud Drive、Git、或端到端加密同步。当前阶段仍以本地优先和单机稳定为主。

## 执行规则

每轮优化必须满足：

1. 有明确用户价值，不做纯内部重构。
2. 代码改动有测试覆盖；前端改动至少通过类型检查。
3. 发布路径只使用 `make package-mac`，不使用 Tauri 发布。
4. 验证通过后推送 GitHub，并创建 Release，让旧版本可直接更新。
5. 版本号按语义化版本递增；新增功能升 minor，修复或小增强升 patch。

## 下一步建议

当前最高优先级是 P0-1 检索性能与资料可发现性。完成后建议继续 P0-2 数据备份与恢复，因为它直接影响长期使用的安全感，也是后续自动同步、自动更新、批量整理的基础保险。
