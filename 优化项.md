### 模块一：工作区上下文隔离 (Workspace Isolation)
目标：实现类似浏览器 Space 的多工作区硬隔离。
1. 数据库与类型演进：
   - 在 `backend/src/trace_api/schema.sql` 中新增表 `workspace` (id TEXT PK, name TEXT, theme_color TEXT, default_llm_profile_id TEXT FK)。
   - 在现有的 `thread`, `note`, `report`, `source` 表中，全部追加 `workspace_id TEXT` 字段。
   - 提供兼容现有本地数据的迁移逻辑（如在启动时执行 ALTER TABLE 并注入默认 Workspace）。
   - 同步更新后端的 Pydantic schemas 和前端的 `frontend/src/lib/types.ts`。
2. 后端 API 改造：
   - 为 Threads, Notes, Todos, Reports, Inbox 的所有查询和新建接口补充 `workspace_id` 过滤逻辑。
   - 新增 `workspaces.py` 路由处理 Workspace 的 CRUD。
3. 前端交互改造：
   - 在前端建立全局的 Workspace 状态（支持持久化）。
   - 改造 Shell 侧边栏以支持工作区列表渲染。
   - 监听全局键盘事件，支持按 `⌃ 1`, `⌃ 2` (Control + 数字) 极速切换工作区，切换时重置视图数据。

### 模块二：本地外部知识库静默挂载 (Watch Folder)
目标：后台监听外部 Markdown 文件夹（如 Obsidian 库）的变动并同步为素材。
1. 后端监听服务：
   - 在 `backend/src/trace_api/services/` 引入 `watchdog` 构建本地目录监控脚本。
   - 逻辑：监听到 `.md` 文件新增或修改时，计算 sha256 hash，对比 `source` 表中对应文件的 hash。若变动，则读取全文，作为 `kind='file'` 更新或插入 `source` 表，并触发后续的拆句分拣。
2. 前端配置：
   - 在 Settings 页面增加“本地库挂载路径”配置项，允许输入绝对路径，并调用后端启动该目录的监听服务。
   - 将对应 Evidence 的前端溯源链接支持修改为系统本地文件协议（如 file://）。

### 模块三：自定义 AI 汇报编排与局部重写 (⌘J)
目标：支持自定义的报告生成提示词，以及编辑器内的 AI 局部重写。
1. 引擎改造：
   - 新增表 `report_template` (id TEXT PK, name TEXT, system_prompt TEXT, context_variables TEXT)。同步 Pydantic 和 TS 类型。
   - 改造 `backend/src/trace_api/services/llm.py`，允许生成 Report 时注入 Template 的 `system_prompt`。
   - 新增一个 `partial_rewrite` 接口，接收用户选中的文本段落和修改指令，返回流式 SSE 结果。
2. 前端编辑器改造：
   - 在 `frontend/src/components/RichEditor` (基于 TipTap) 中增加快捷键监听。
   - 当用户选中文本并按下 `⌘ J` 时，弹出指令输入框。回车后请求 `partial_rewrite`，以流式打字机效果原地替换选中的文本节点。
